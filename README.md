# MyBudget

Локальний вебзастосунок для денного прогнозування ресурсу на горизонт від кількох днів до кількох місяців. Модель бере наявний бюджет, вільні гроші, наступний дохід з ймовірною датою, ціль резерву у відсотках, щоденні витрати і точкові події, а потім проганяє весь період день за днем.

## Що рахує
- `Базовий` сценарій — ваш денний темп плюс точкові події.
- `Економний` — зрізає щоденні витрати і перекидає невикористану пайку у вільні гроші.
- `Агресивний` — тягне витрати до верхньої межі комфорту і не тримає резерв окремо.
- `Форс-мажор` — посеред горизонту відкушує 20% ресурсу.
- Динамічну пайку: якщо день вийшов дорожчим за безпечний ліміт, наступні ліміти стискаються.
- Коридор між економним та агресивним режимами.
- Ватерлінію: момент, коли сумарний ресурс падає до цільового резерву.
- День, коли операційний бюджет починає їсти вільні гроші.

## Вхідні поля
- `available_budget` — наявний операційний бюджет.
- `free_money` — вільні гроші або подушка.
- `next_income_amount` — сума наступного доходу.
- `next_income_date` — ймовірна дата доходу.
- `monthly_savings_percent` — ціль збереження як відсоток від ресурсу в межах горизонту.
- `daily_expense` — базові щоденні витрати.
- `days` — горизонт прогнозу в днях.
- `events_text` — точкові удари по бюджету, по одному рядку.

Формат `events_text`:

```text
6;1400;Ліки
13;3200;Ремонт
24;1800;Свято
```

Можна використовувати і `;`, і `|`.

## Локальні файли
- `storage/budget.json` — останній введений бюджет.
- `storage/history.json` — історія ручних запусків.

## Запуск
Тепер запускати потрібно з папки застосунку:

```powershell
cd .\Mybuget
python .\app.py
```

або

```powershell
cd .\Mybuget
.\run.bat
```

`run.bat` всередині папки використовує локальну `.venv`.

## HTTP-маршрути
- `GET /` — HTML-інтерфейс.
- `GET /state` — поточний `budget.json`, історія і попередньо порахований forecast.
- `POST /calculate` — новий розрахунок. Параметр `save_history: false` рахує без дубля в історії.
- `DELETE /history` — очищення `history.json`.
